一 SpringMvc简介

1. SpringMvc中重要组件

SpringMvc有4个重要的组件：

  1. DispatcherServlet
  2. HandlerMapping
  3. HandlerAdapter
  4. ViewResovler

    DispatcherServlet

    DispatcherServlet是个前端控制器，提供Spring Web MVC的集中访问点（接收所有请求），然后将请求分配给指定的组件处理。

    HandlerMapping

    解析请求格式，判断希望要执行的那个具体的方法

    HandlerAdapter

    负责调用具体的方法

    ViewResovler

    视图解析器，解析结果，准备跳转到具体的物理视图。

    2. SpringMvc运行原理图


    所有的请求都先经过DispatcherServlet进行分配给指定的HandlerMapping，之后HandlerMapping找到请求对应的方法，然后找到方法对应的HandlerAdapter，HandlerAdapter处理完后返回给控制器然后跳转到具体的视图

    3. Spring容器和SpringMvc容器是父子容器

    查看源码可知：两个容器间的关系

    由于SpringMvc容器和Spring容器是父子关系，所以在SpringMvc容器能调用Spring容器的所有内容

二 搭建一个简单的SpringMvc Demo

1. 导入所需要的jar包

2. 在web.xml中配置前端控制器DispatcherServlet

如果不配置会找/WEB-INF/-servler.xml

<servlet>
    <servlet-name>llj</servlet-name>
    <servlet-class>org.springframework.web.servlet.DispatcherServlet</servlet-class>
    <init-param>
        <param-name>contextConfigLocation</param-name>
        <param-value>classPath:springmvc.xml</param-value>
    </init-param>
    <load-on-startup>1</load-on-startup>
</servlet>
<servlet-mapping>
    <servlet-name>llj</servlet-name>    
    <url-pattern>/</url-pattern>
</servlet-mapping>

3. 在src下新建springmvc.xml

springmvc.xml文件名称需要与classpath:后面的值相同

<init-param>
    <param-name>contextConfigLocation</param-name>
    <param-value>classpath:springmvc.xml</param-value>
</init-param>

springmvc.xml配置内容如下

<?xml version="1.0" encoding="UTF-8"?>
    <beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
    xmlns:mvc="http://www.springframework.org/schema/mvc"
    xmlns:context="http://www.springframework.org/schema/context"
    xsi:schemaLocation="http://www.springframework.org/schema/beans
        http://www.springframework.org/schema/beans/spring-beans.xsd
        http://www.springframework.org/schema/context
        http://www.springframework.org/schema/context/spring-context.xsd
        http://www.springframework.org/schema/mvc
        http://www.springframework.org/schema/mvc/spring-mvc.xsd">

    <!-- 扫描注解 -->
    <context:component-scan base-package="com.bjsxt.controller"></context:component-scan>
    <!-- 注解驱动 -->
    <mvc:annotation-driven>
    </mvc:annotation-driven>
    <!-- 静态资源  两个*包括所有子文件及文件夹 不做静态资源的放行SpringMvc会把它当Servlet来处理-->
    <!-- mapping请求的url -->
    <mvc:resources location="/js/ " mapping="/js/**"></mvc:resources>
    <mvc:resources location="/css/ " mapping="/css/**"></mvc:resources>
    <mvc:resources location="/imges/ " mapping="/imges/**"></mvc:resources>
</beans>

4. 编写控制类

@Controller
public class DemoController {

    @RequestMapping("demo")
    public String demo() {
        System.out.println("执行了demo");
        return "/main.jsp";
    }

    @RequestMapping("demo2")
    public String  demo2() {
        System.out.println("执行了demo2");
        return "main2.jsp";
    }
}

6. 测试

访问以下路径http://localhost:8080/youProjectName/demo2
测试demo–控制台打印消息并跳转到main.jsp

测试demo2–控制台打印消息并跳转到main2.jsp

7. 存在中文乱码问题需要配置过滤器

Web.xml中配置一个filter

<!-- 字符编码过滤器 -->
<filter>
    <filter-name>encoding</filter-name>
    <filter-class>org.springframework.web.filter.CharacterEncodingFilter</filter-class>
    <init-param>
        <param-name>encoding</param-name>
        <param-value>utf-8</param-value>
    </init-param>
</filter>
<filter-mapping>
    <filter-name>encoding</filter-name>
    <url-pattern>/*</url-pattern>
</filter-mapping>
