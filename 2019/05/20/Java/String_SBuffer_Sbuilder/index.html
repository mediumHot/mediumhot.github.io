String、StringBuffer和StringBuilder区别

发表于 2019-05-20 | 分类于 Java

三者运算性能比较####

通常情况下：StringBuilder > StringBuffer > String

String:

由于String是final修饰的，所以它的值是不可变的，这就导致每次对String的操作都会生成新的String对象，这就造成了效率低下，而且会浪费大量的内存空间。

StringBuffer和StringBuilder:

StringBuffer和StringBuilder都是用来构造可变的字符串，主要通过append()的方法进行字符串的拼接。

两者之间的主要区别在于，StringBuffer的所有公开方法都是synchronized进行修饰的，这就导致了StringBuilder的性能会优于StringBuffer。

总结

String：适用于少量的字符串操作。

StringBuffer：适用于多线程下字符串操作操作。

StringBuilder：适用于单线程下字符串操作操作。

以上就是String、StringBuffer和StringBuilder区别，感谢大家的阅读。
